## 📄 Overview

<a href="#expanDrive">ExpanDrive</a> mounts cloud storage services and remote servers as native drives in Finder — making Google Drive, Dropbox, OneDrive, Amazon S3, SFTP servers, and other remote storage accessible as if they were local drives connected to the Mac, without requiring files to sync locally first.

The <a href="#expanDrive">native drive mounting</a> makes remote storage appear in Finder's sidebar alongside local drives — clicking the mounted cloud drive opens it in Finder showing the remote files and folders. Any application that can open or save files can work with files on the mounted drive directly, without the user needing to download files first or use a specific cloud service application. Opening a document from an S3 bucket or an SFTP server works the same way as opening from a local drive — navigate in Finder, double-click the file, the application opens it. The <a href="#expanDrive">multi-service coverage</a> supports Google Drive, Dropbox, OneDrive, Box, Amazon S3, Backblaze B2, SFTP, FTP, WebDAV, and many other storage services and protocols — connecting to any of them through the same ExpanDrive interface and mounting each as its own drive in Finder. Users with storage across multiple services manage all of them through one interface.

The <a href="#expanDrive">direct file access without local sync</a> is ExpanDrive's primary advantage over native cloud sync clients — files on the mounted drive stream on demand rather than occupying local storage. A team member with 500GB of project files on Google Drive doesn't need 500GB of local storage — they open files from the mounted ExpanDrive drive and only the actively opened file transfers. The <a href="#expanDrive">any application compatibility</a> extends cloud file access to applications that don't have native cloud integration — opening cloud files in Final Cut Pro, Photoshop, Logic Pro, or any other application that uses standard file open dialogs, without requiring each application to individually support the cloud service.

<a href="#expanDrive">SFTP and remote server support</a> connects to remote Linux and development servers via SFTP, making server file systems accessible in Finder alongside cloud storage — developers editing server files directly from desktop applications without separate SCP or FTP transfers. <a href="#expanDrive">Automatic reconnection</a> re-establishes connections to mounted drives when network connectivity returns after interruption — maintaining persistent drive availability without manual reconnection.

<a href="#expanDrive">File transfer with progress indication</a> shows upload and download progress for file operations on mounted drives — visibility into transfer status for large file operations. <a href="#expanDrive">Strong Sync selective sync mode</a> in some configurations provides selective local caching of specific folders for offline access — combining on-demand streaming with offline availability for designated folders.

## 🤔 FAQ

<details>
<summary>What cloud services does ExpanDrive support?</summary>

Google Drive, Dropbox, OneDrive, Box, Amazon S3, Backblaze B2, SFTP, FTP, WebDAV, and many others. Check the current ExpanDrive website for the complete supported service list.

</details>

<details>
<summary>Does ExpanDrive sync files locally?</summary>

By default, ExpanDrive streams files on demand without local sync — files aren't duplicated to local storage. Some configurations provide selective local caching for offline access.

</details>

<details>
<summary>Can I access ExpanDrive-mounted drives from any application?</summary>

Yes. Mounted drives appear as standard Finder drives accessible from any application that uses standard file open and save dialogs.

</details>

<details>
<summary>Does ExpanDrive support SFTP for remote servers?</summary>

Yes. SFTP connections mount remote Linux and development server file systems as Finder drives.

</details>

<details>
<summary>Does ExpanDrive automatically reconnect after network interruptions?</summary>

Yes. Automatic reconnection re-establishes connections when network connectivity returns.

</details>

<details>
<summary>Can ExpanDrive mount multiple cloud services simultaneously?</summary>

Yes. Multiple services mount as separate drives in Finder simultaneously.

</details>

<details>
<summary>Does ExpanDrive mount automatically at Mac startup?</summary>

Yes. Drives configure to mount automatically when the Mac starts.

</details>

<details>
<summary>Is ExpanDrive a subscription?</summary>

Yes. ExpanDrive uses subscription pricing. Check the ExpanDrive website for current pricing.

</details>

<details>
<summary>Does ExpanDrive run on Apple Silicon?</summary>

Yes. Current versions support Apple Silicon Macs natively.

</details>
